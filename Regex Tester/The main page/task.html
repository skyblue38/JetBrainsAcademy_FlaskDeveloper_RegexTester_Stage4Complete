Description

Wonderful! We have a database to store the tests, now, how can we improve the program? Let's add a page with a form to get input from users. With that simple solution, anyone will be able to try out their knowledge.

Objectives

You can always build a simple web page with HTML/CSS/JS. Pages that do not change information on them are called static. For example, if you need to create a simple one-page portfolio to store something about yourself, you can create an HTML document, and voila! It will be displayed in a browser. But this solution isn't perfect for blogs because it will be awkward to create a page for each post. However, you can create dynamic pages with templates. In our application, we will have a couple of pages that don't change their layout, but the content there changes constantly. You don't want to write every page yourself â€” that's where templates become handy.

This time, the task is to create a home page as an HTML template and connect a Flask View to it. The template should be of a simple layout:

  • At the top of the page, place a greeting message: <h2> Welcome to regex testing tool! </h2>;
  • Next, create the main form with two text fields and a submit button. For your input tag, use the name=... attribute; you will access the form data using these names. Please, use regex and text for the name attributes;
  • At the bottom, add the link to the history page: <a href="/history/"> History </a>.

The main page needs to be available under a GET request to localhost:5000. Create a view that renders a template for it. To complete that, use a Flask function: render_template(template_name).

Don't forget that templates must be stored in the templates/ folder.
